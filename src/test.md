# test 技术总结

## 职责定位

`test/` 目录主要存放端到端和联调脚本，偏“可执行验证”，而不是 Django 单元测试。

## 主要脚本

| 文件 | 作用 |
| --- | --- |
| `e2e_pipeline.py` | 端到端数据采集翻译流水线验证 |
| `e2e_ingest_key_push.py` | 校验推送链路与 ingest key 鉴权 |
| `create_test_users.py` | 构造测试用户数据 |

## 技术要点

- 此目录更接近“集成测试工具箱”，适合联调和回归场景。
- 涉及采集推送、账号准备等真实运行链路，是发现环境配置问题的重要补充。
