# docs 技术总结

## 职责定位

`docs/` 是项目的设计资产库，集中存放系统设计、任务拆分、API 文档、调试手册和接入说明。

## 文档覆盖范围

| 主题 | 代表文档 |
| --- | --- |
| 采集任务管理 | `采集任务管理系统设计.md`, `采集任务管理API接口文档.md` |
| 翻译管理 | `翻译管理系统设计.md`, `翻译管理API接口文档.md` |
| 导入导出 | `导入导出系统设计.md`, `导入导出API接口文档.md` |
| 数据采集翻译入库 | `数据采集翻译入库实现方案.md`, `数据采集翻译入库功能清单.md` |
| 爬虫接入 | `爬虫推送接入说明.md`, `境外网站爬虫接入系统设计.md` |
| 监控与调试 | `数据统计监控系统设计.md`, `数据统计监控调试手册.md` |
| AI 相关 | `业务AI剥离系统设计.md` |

## 技术要点

- 文档密度较高，说明项目具备较完整的方案设计和任务拆分过程。
- 文档目录同时覆盖“设计、接口、实现方案、任务拆分”，适合作为交付与维护依据。
- 对外接入和内部模块演进都在此有痕迹，是理解历史决策的重要入口。
